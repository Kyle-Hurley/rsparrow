Getting Started with rsparrow

Overview

rsparrow is an R package for executing and evaluating SPARROW (SPAtially Referenced Regressions On Watershed attributes) models. SPARROW models three watershed processes:

  1. Source generation — contaminant inputs from land uses (e.g., agricultural N, atmospheric deposition)
  2. Land-to-water delivery — transport to streams via surficial and subsurface pathways, modulated by climate and landscape variables
  3. In-stream transport and decay — removal and attenuation as water moves downstream through reaches and reservoirs

The package implements static (long-term mean annual) SPARROW models. It generates reach-level load and yield predictions, source attribution, parameter uncertainty via bootstrap, and diagnostic plots.

Installation

install.packages("rsparrow")

# Or the development version from GitHub:
# remotes::install_github("Kyle-Hurley/rsparrow")

Step 1: Load the Example Data

rsparrow includes a synthetic 60-reach example dataset, sparrow_example, that demonstrates the full modeling workflow without requiring real data.

library(rsparrow)

str(sparrow_example, max.level = 1)
#> List of 5
#>  $ reaches        :'data.frame': 60 obs. of  20 variables:
#>  $ sites          :'data.frame': 5 obs. of  6 variables:
#>  $ parameters     :'data.frame': 2 obs. of  8 variables:
#>  $ design_matrix  :'data.frame': 1 obs. of  2 variables:
#>  $ data_dictionary:'data.frame': 20 obs. of  5 variables:

The dataset contains five elements:

  • reaches — 60-reach dendritic network with two tributary branches draining to a single outlet. Columns include topology (waterid, fnode, tnode), drainage area (demiarea, demtarea), a synthetic agricultural-N source (agN), and a precipitation delivery variable (ppt).
  • sites — 5 monitoring sites with synthetic observed loads (depvar, kg/year).
  • parameters — parameter configuration (SOURCE and DELIVF types).
  • design_matrix — binary matrix associating sources with delivery variables.
  • data_dictionary — optional; maps internal SPARROW variable names to data column names (not needed when column names already match).

Step 2: Inspect the Network Topology

Before fitting a model, compute the hydrological sequence — the reach ordering from upstream to downstream that SPARROW uses for load accumulation.

network <- rsparrow_hydseq(sparrow_example$reaches)
head(network[order(network$hydseq), c("waterid", "fnode", "tnode", "hydseq")])
#>    waterid fnode tnode hydseq
#> 36      36   301   302    -60
#> 26      26   201   202    -59
#> 11      11   101   102    -58
#> 1        1     1     2    -57
#> 37      37   302   303    -56
#> 27      27   202   203    -55

Reaches with more negative hydseq values are further downstream. The outlet reach has the most negative value.

Step 3: Fit the Model

Quick Start (one-call interface)

Pass the three required data frames to rsparrow_model(). No CSV files, directories, or data dictionary are required when column names already match SPARROW variable names:

model <- rsparrow_model(
  sparrow_example$reaches,
  sparrow_example$parameters,
  sparrow_example$design_matrix
)

Composable Workflow

For finer control, call the two steps separately. This lets you inspect and reuse the prepared data object, or swap out estimation options without re-validating inputs:

# Step 1: Validate and prepare inputs
sparrow_data <- rsparrow_prepare(
  sparrow_example$reaches,
  sparrow_example$parameters,
  sparrow_example$design_matrix
)
print(sparrow_data)
#> rsparrow_data
#>   Reaches     : 60
#>   Calibration sites (depvar > 0 & calsites == 1): 5
#>   Parameters  : 2 (1 SOURCE, 1 DELIVF)

# Step 2: Estimate the model
model <- rsparrow_estimate(sparrow_data)

# Step 3: Predict at all reaches (already done when if_predict = TRUE)
predictions <- predict(model)

# Post-processing (same functions as before)
model <- rsparrow_bootstrap(model, n_boot = 200, seed = 42)
model <- rsparrow_validate(model)

Step 4: Examine Results

The returned rsparrow object supports standard S3 methods:

# Compact summary
print(model)

# Full estimation summary
summary(model)

# Parameter estimates
coef(model)
#>        agN        ppt
#>  3.4176761 -0.0015935

# Log residuals at calibration sites
residuals(model)

# Parameter covariance matrix
vcov(model)

# Predictions (populated when if_predict = TRUE)
names(model$predictions)
#> [1] "oparmlist"          "loadunits"          "predmatrix"
#> [4] "oyieldlist"         "yieldunits"         "yldmatrix"

Step 5: Diagnostic Plots

Diagnostic plots use base R graphics and are available for any fitted model:

plot(model, type = "residuals")           # observed vs predicted, residuals
plot(model, type = "residuals", panel = "B")  # Q-Q, boxplots
plot(model, type = "sensitivity")         # parameter sensitivity
plot(model, type = "spatial")             # spatial autocorrelation

Step 6: Bootstrap Uncertainty

Estimate parameter uncertainty via parametric bootstrap:

model <- rsparrow_bootstrap(model, n_boot = 200, seed = 42)
model$bootstrap$bEstimate  # bootstrapped parameter distributions

Step 7: Scenario Analysis

Evaluate the effect of changing source inputs:

# Simulate a 50% reduction in agricultural N loading
model <- rsparrow_scenario(model, source_changes = list(agN = 0.5))

API Reference

Function Description
rsparrow_prepare() Validate inputs; return rsparrow_data object
rsparrow_estimate() Fit NLLS model; return rsparrow object
rsparrow_model() One-call convenience wrapper for the two steps above
predict() Reach-level load/yield predictions
rsparrow_bootstrap() Parametric bootstrap for parameter uncertainty
rsparrow_validate() Holdout validation
rsparrow_scenario() Source-change scenario analysis
rsparrow_hydseq() Compute hydrological sequence
write_rsparrow_results() Write results to disk
